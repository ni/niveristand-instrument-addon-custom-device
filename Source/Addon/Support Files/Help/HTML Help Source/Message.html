Message

This section contains the items for defining a message. If the message is a write only, configure the Command and delete the Response. If the message is a read only, configure the Response and delete the Command. If the message is a query (write then read), configure both.

What do you want to do?

Set Generation Type

Currently three message generation types are supported: Trigger Only, Periodic & Trigger, and Value Change & Trigger.

Trigger Only fits for single event messages. As an example, setting the instrument configuration at the beginning of a test.

Periodic & Trigger has the same triggerable function as Trigger Only, but will automatically execute the message after a set period. So, if you have a period of 1 second and don't trigger the message, it will automatically execute by itself. If you trigger the message, the periodic timers get reset.

Note: The starting and stopping of the periodic generation can be controlled through the Enable Periodic channel.

Value Change & Trigger will trigger the message by using the Trigger Channel or anytime a VeriStand channel used in the command field is changed. If the command message contains multiple VeriStand channels, the message will be sent each time one of the channels has changed.

Note: For absolute control when using Value Change & Trigger with multiple VeriStand channels in a command, Trigger Only generation type is recommended.

Select the generation type from the Generation Type field. Trigger Only will be the most common. If you are selecting Periodic & Trigger, set the Period (ms) to the desired setting.

Note: Limit the number of periodic messages as much as possible on slower protocols and baud rates as this could delay other messages.

Automatically Include Message Protection

Message protections are often used to make sure the data sent and received match. Common protection schemes include checksums or CRCs. To enable this, check the Include Protection field and select the desired type. The protection will be automatically calculated for you and placed at the end of the message.
