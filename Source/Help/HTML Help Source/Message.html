Message

This section contains the items for defining a message. If the message is a write only, configure Command and delete the Response. If the message is a read only, configure the Response and delete the Command. If the message is a query (write then read) configure both.

What do you want to do?

Set Generation Type

There are two types of generation that are currently supported. One is a Trigger. Triggers are mainly for single event messages. As an example, setting the instrument configuration at the beginning of a test. The other is Periodic & Trigger. This has the same triggerable function but will automatically execute the message after a set period. So, if you have a period of 1 second and don't trigger the message, it will automatically execute by itself. If you trigger the message, the periodic timers get reset.

Note: The starting and stopping of the periodic generation can be controlled through the Enable Periodic channel.

Select the generation type from the Generation Type field. Trigger will be the most common. If you are sellecting Periodic & Trigger, set the Period (ms) to the desired setting.

Note: Limit the number of periodic messages as much as possible on slower protocols and baud rates as this could delay other messages.

Automatically Include Message Protection

Message protections are often used to make sure the data sent and received match. Common protection schemes include checksums or CRCs. To enable this, check the Include Protection field and select the desired type. The protection will be automatically calculated for you and placed at the end of the message.
